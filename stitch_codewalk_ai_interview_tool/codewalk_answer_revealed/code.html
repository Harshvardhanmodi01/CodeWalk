import { useMemo } from 'react';

const DataGrid = ({ items, filter }) => {
  const filteredData = useMemo(() => {
    // Heavy computation here
    return items.filter(item => item.includes(filter));
  }, [items]); /* BUG DETECTED */

  return (
    <div>
      {filteredData.map(i => <span>{i}</span>)}
    </div>
  );
};

Performance Analysis

The useMemo hook above is missing a critical dependency. Identify the missing variable in the dependency array and explain why its absence causes stale UI states.

LIVE ASSESSMENT
check_circle Expected Answer

The missing dependency is filter.

In React's useMemo hook, the dependency array tells React when to re-calculate the memoized value. Currently, the array only contains [items]. This means that even if the filter string changes, the component will skip the expensive recalculation and return the previously cached filteredData from the last time items changed.

// Correct implementation:
const filteredData = useMemo(() => {
  return items.filter(item => item.includes(filter));
}, [items, filter]);

By adding both variables, you ensure the computation is stable relative to all its external inputs, preventing "stale" renders where the UI doesn't reflect the current user input.
